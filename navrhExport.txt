Uživatel upraví model ve webové aplikaci (pozice prvků, metadata, přidaný nábytek, smazání).
Klikne na tlačítko Export IFC.
Backend vezme původní IFC + uložené změny a vygeneruje nový IFC soubor ke stažení.
Výsledný IFC půjde otevřít i v jiném BIM nástroji.


Spring Boot bude export řídit.
Samotnou IFC manipulaci udělá specializovaný exporter (IfcOpenShell), který backend zavolá.
IfcOpenShell má jádro v C++ a nejběžněji se používá přes Python API, 
takže pro čistě Java backend to typicky znamená volat ho jako externí proces/službu (např. Python sidecar) 
a Java část nechat jen řídit workflow exportu.


IfcRelContainedInSpatialStructure - ifcfurniture, ifcfurnishingelement, IfcElectricAppliance, IfcDistributionControlElement, IfcFlowTerminal, IfcSanitaryTerminal

Přes IfcRelContainedInSpatialStructure nelze linkovat věci, které nejsou IfcProduct (např. IfcPropertySet, IfcMaterial, typy IfcTypeObject, osoby, úkoly).
Jeden prvek nemůže mít současně dvě „hlavní“ containment vazby (tj. být najednou pod dvěma IfcSpace nebo pod IfcSpace i IfcBuildingStorey).
Prvky přes více místností (typicky stěna) se správně neřeší jako „patří do více IfcSpace“ přes containment, ale přes prostorové hranice/reference (IfcRelSpaceBoundary / IfcRelReferencedInSpatialStructure).
Takže: IfcSpace = hlavně fyzické produkty v místnosti, ne datové/definiční objekty a ne vícenásobné primární umístění.
